# Prebuilt Libraries

This directory contains prebuilt static libraries for each supported platform.
These are automatically generated by the GitHub Actions workflow when changes
are made to the Zig source code.

## Directory Structure

```
lib/
├── darwin_arm64/    # macOS Apple Silicon
│   ├── galleon.h
│   └── libgalleon.a
├── darwin_amd64/    # macOS Intel
│   ├── galleon.h
│   └── libgalleon.a
├── linux_arm64/     # Linux ARM64
│   ├── galleon.h
│   └── libgalleon.a
├── linux_amd64/     # Linux x86_64
│   ├── galleon.h
│   └── libgalleon.a
└── windows_amd64/   # Windows x86_64
    ├── galleon.h
    └── libgalleon.a
```

## For Users

If you're installing Galleon via `go get`, the prebuilt libraries are used
automatically. No additional setup is required.

```bash
go get github.com/NerdMeNot/galleon/go
```

## For Developers

If you're developing Galleon and want to build the Zig library locally:

1. Install Zig 0.13+
2. Build the library:
   ```bash
   cd core
   zig build -Doptimize=ReleaseFast
   ```
3. Build Go with the `dev` tag:
   ```bash
   go build -tags dev ./...
   go test -tags dev ./...
   ```

## Rebuilding Prebuilt Libraries

The GitHub Actions workflow (`build-libs.yml`) automatically rebuilds these
libraries when:
- Changes are pushed to `core/src/**`
- Changes are pushed to `core/build.zig`
- The workflow is manually triggered

You can also manually trigger the workflow from the GitHub Actions tab.
